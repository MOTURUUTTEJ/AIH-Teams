# Contributing to AIH-Teams

Thank you for contributing. This repository follows a fork-and-pull-request workflow to keep collaboration clear and professional.

## Workflow: Fork and Pull Request

1. Fork `MOTURUUTTEJ/AIH-Teams`
2. Clone your fork locally
3. Create a focused branch:
   ```bash
   git checkout -b feat/<short-description>
   ```
4. Make your changes
5. Commit using clear messages:
   ```bash
   git commit -m "feat: add <what you changed>"
   ```
6. Push your branch and open a pull request

## Code and Content Standards

- Keep changes focused on one clear objective
- Use readable naming and consistent formatting
- Update documentation when behavior or structure changes
- Do not commit secrets, credentials, or generated dependency folders

## Submission Expectations

When opening a pull request, include:
- A short summary of the contribution
- Why the change is needed
- Any relevant screenshots or evidence (if UI/content related)
- Follow-up tasks, if any

## Review Process

- Maintainers review for clarity, scope, and project alignment
- You may receive requested changes; address them in the same PR
- Contributions are merged after approval and successful checks

## Professional Collaboration Notes

- Be respectful and constructive in review discussions
- Prefer incremental, reviewable pull requests over large drops
- If unsure about direction, open a draft PR early for feedback
